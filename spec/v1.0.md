## 1. Overview

### 1.1 Objectives

This document defines the OpenAIX (AI Experience) scoring protocol standard for quantifying how AI-friendly a website is for AI Agents (LLMs, RAG systems, crawlers).

### 1.2 Scope

This specification covers:
- Definitions and calculation methods for four scoring dimensions
- Classification standards for scoring grades (Class S/A/B/C)
- Developer optimization guidelines
- Testing and validation methods

### 1.3 Terminology

| Term | Definition |
|------|------------|
| **AIX** | AI Experience - Metric measuring AI-friendliness of websites |
| **SNR** | Signal-to-Noise Ratio - Content density metric |
| **LLM** | Large Language Model |
| **RAG** | Retrieval-Augmented Generation |
| **JSON-LD** | JSON for Linked Data - Structured data format |
| **llms.txt** | AI-specific site description file, similar to robots.txt |

## 2. Scoring System

### 2.1 Overall Architecture

The OpenAIX score is calculated from four dimensions with weighted contributions:

```
Total Score = SNR × 30% + Semantic × 30% + Token Economy × 20% + Permissions × 20%
```

### 2.2 Scoring Grades

| Grade | Score Range | Description | Recommendation |
|-------|-------------|-------------|----------------|
| **S** | 85-100 | Silicon Native | Excellent AI experience, fully agent-optimized |
| **A** | 70-84 | Agent Friendly | Good AI experience, clear structure, low noise |
| **B** | 50-69 | Acceptable | Usable but has room for improvement |
| **C** | < 50 | Needs Improvement | Significant AI access barriers exist |

---

## 3. Dimension 1: SNR (Signal-to-Noise Ratio)

### 3.1 Definition

SNR measures the ratio of meaningful content to total content in HTML.

```
SNR = (Clean Tokens / Raw Tokens) × 100%
```

- **Raw Tokens**: Token count of raw HTML
- **Clean Tokens**: Token count after removing scripts/styles, converted to Markdown

### 3.2 Scoring Criteria

| SNR Range | Score | Rating |
|-----------|-------|--------|
| ≥ 40% | 100 | Excellent |
| 20% - 40% | 70-100 | Good (linear interpolation) |
| 10% - 20% | 50-70 | Acceptable (linear interpolation) |
| 5% - 10% | 30-50 | Needs Improvement (linear interpolation) |
| < 5% | 0-30 | Poor (linear interpolation) |

### 3.3 Calculation Method

1. Calculate Raw Tokens using `tiktoken` (cl100k_base)
2. Remove the following tags using BeautifulSoup:
   - script, style, noscript
   - iframe, canvas, svg
   - template, embed, object
3. Convert to Markdown using markdownify
4. Recalculate Clean Tokens

### 3.4 Optimization Recommendations

- Reduce unnecessary HTML wrapper layers
- Remove unused CSS/JS
- Avoid inline styles and scripts

## 4. Dimension 2: Semantic Structure

### 4.1 Definition

Evaluates the semantic quality of HTML, including tag usage, structured data, and metadata quality.

### 4.2 Score Composition

Total = Semantic Tags(25) + Heading Hierarchy(25) + JSON-LD(25) + Image Accessibility(10) + Metadata(15)

#### 4.2.1 Semantic Tags (25 points)

Supported semantic tags: article, nav, header, main, section, aside, footer

```
Score = (Used Tags / 7) × 25
```

#### 4.2.2 Heading Hierarchy (25 points)

- H1 present: +15 points
- 2+ heading levels: +10 points

#### 4.2.3 JSON-LD Structured Data (25 points)

Scoring criteria:
- @context present: +5 points
- @type present: +5 points
- name/description/url present (at least one): +5 points
- Content length > 500 characters: +10 points

#### 4.2.4 Image Accessibility (10 points)

```
Score = (Images with alt / Total Images) × 10
```

#### 4.2.5 Metadata Quality (15 points)

- Title tag present: +8 points
- Meta description present: +7 points

### 4.3 Hidden Gem Rule

For visually rich sites with complete structured data (e.g., Apple, Shopify), if they meet:
- Text density < 0.15
- JSON-LD score ≥ 15
- Has H1 tag
- SNR < 40%

Then this dimension has a minimum score of 85.

## 5. Dimension 3: Token Economy

### 5.1 Definition

Evaluates the cost efficiency of AI reading the page.

### 5.2 Scoring Criteria

Based on Clean Tokens count:

| Tokens | Cost Rating | Score |
|--------|-------------|-------|
| < 2,000 | Low | 100 |
| 2,000 - 6,000 | Medium | 85 |
| 6,000 - 15,000 | High | 65 |
| > 15,000 | Very High | 40 |

### 5.3 Cost Estimation

```
Estimated Cost (USD) = (Clean Tokens / 1000) × $0.03
```

Based on GPT-4 pricing model.

## 6. Dimension 4: Agent Permissions

### 6.1 Definition

Evaluates website accessibility and friendliness to AI Agents.

### 6.2 Score Composition

Starting score: 100

#### 6.2.1 Robots.txt Check (-10 points/Agent)

AI Agents checked:
- GPTBot (OpenAI)
- CCBot (Common Crawl)
- anthropic-ai (Anthropic)
- ClaudeBot (Anthropic)
- ChatGPT-User (OpenAI)
- Google-Extended (Google)
- PerplexityBot (Perplexity)

Each Disallowed Agent deducts 10 points.

#### 6.2.2 Response Time (-5 to -10 points)

- > 3 seconds: -10 points
- 1.5 - 3 seconds: -5 points

#### 6.2.3 llms.txt Bonus (+80 minimum)

If `/llms.txt` or `/.well-known/llms.txt` is detected, this dimension score is at least 80.

### 6.3 llms.txt Specification

#### 6.3.1 File Location

- Root directory: `/llms.txt`
- Alternative: `/.well-known/llms.txt`

#### 6.3.2 Format Example

```
# OpenAIX Site Description

## Overview
This website provides [description].

## Key Sections
- /docs: API documentation
- /blog: Technical articles
- /products: Product catalog

## Access Rules
- Allow: All AI agents
- Rate limit: 1 request per second

## Contact
For questions: ai-access@example.com
```

## 7. Testing and Validation

### 7.1 Local Testing

```bash
# Install
pip install -e .

# Test single URL
python -m openaix https://example.com

# Batch testing
python benchmark.py --urls-file urls.txt
```

### 7.2 CI/CD Integration

```yaml
# .github/workflows/aix-check.yml
name: AIX Score Check
on: [push]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Install OpenAIX
        run: pip install openaix-scorer
      - name: Check Score
        run: |
          python -c "
          from openaix import OpenAIXScorer
          scorer = OpenAIXScorer()
          result = scorer.score('https://your-site.com')
          assert result['score'] >= 70, f'Score too low: {result[\"score\"]}'
          print(f'✅ AIX Score: {result[\"score\"]}')
          "
```
